REST API: Γενικές αρχές
Το σύστημα λογισμικού που θα αναπτύξετε θα πρέπει να υποστηρίζει ένα RESTful Application Programming
Interface (API) για την αποθήκευση και ανάκτηση των δεδομένων που αποθηκεύονται στη Βάση που θα
υλοποιήσετε. Οι λειτουργίες (κλήσεις) που περιγράφονται ακολούθως πρέπει να υλοποιηθούν όλες,
ανεξάρτητα από το αν θα χρησιμοποιηθούν στις περιπτώσεις χρήσης που θα επιλέξετε.

Documentation
Το API documentation θα πρέπει να ακολουθεί το πρότυπο OpenAPI 3/swagger και να είναι διαθέσιμο στον
φάκελο /documentation του GitHub.

Base URL
Το API θα είναι διαθέσιμο στο ακόλουθο base URL:
https://{{host}}:9876/api

όπου {{host}} είναι η διεύθυνση του server που θα εκτελείται το API.
Τα επιμέρους resources (REST endpoints) που θα διατίθενται μέσω του API θα είναι προσβάσιμα μέσω του
base URL, ως εξής:
{baseURL}/{service}/{path-to-resource}

Όπου {service} μία από τις υπηρεσίες που θα διατίθεται, όπως αναφέρονται παρακάτω. Για παράδειγμα, το
endpoint για την ανάκτηση της τρέχουσας κατάστασης του φορτιστή 123 είναι το
https://localhost:9876/api/point/123

Μορφότυποι δεδομένων
Το API θα επιστρέφει δεδομένα σε κωδικοποίηση UTF8, στους μορφότυπους JSON (content-type:
application/json) και CSV (content-type: text/csv). Ο μορφότυπος csv υποστηρίζεται μόνο για τα
endpoints που επιστρέφουν λίστες. Στην περίπτωση αυτή, η επιλογή του μορφότυπου θα καθορίζεται στην
αίτηση με query parameter ως εξής:
{baseURL}/{service}/{path-to-resource}?format={json|csv}

Αν η παράμετρος format δεν παρέχεται η default τιμή θα είναι json για το API και csv για το CLI.
Ο διαχωριστικός χαρακτήρας (delimiter) στα αρχεία csv θα είναι ο χαρακτήρας ",".

Διαχείριση
Το back-end σας θα υποστηρίζει τα παρακάτω 3 διαχειριστικά endpoints, τα οποία θα υποστηρίξουν τον
αυτοματοποιημένο έλεγχο κατά την παρουσίαση της εργασίας:
1. /admin/healthcheck: Υποστηρίζει τη μέθοδο GET και επιβεβαιώνει τη συνδεσιμότητα (end-to-end
connectivity) μεταξύ του χρήστη και της βάσης δεδομένων. Σε περίπτωση επιτυχούς σύνδεσης
επιστρέφεται κωδικός http status 200 και body που περιέχει το object:
{ "status":"OK",
"dbconnection": [connection string],
"n_charge_points": [number of charge points],
"n_charge_points_online": [number of charge points online],
"n_charge_points_offline": [number of charge points online]
}

Διαφορετικά επιστρέφεται http status code 400 και body που περιέχει ένα αντικείμενο error log.
2. /admin/resetpoints: Υποστηρίζει τη μέθοδο POST και προβαίνει σε αρχικοποίηση του πίνακα
σημείων φόρτισης με τις τιμές που περιέχονται στο αρχείο json με ενδεικτικά δεδομένα σημείων
φόρτισης, το οποίο υπάρχει για κατέβασμα στο φάκελο της εργασίας στο helios. Στο περιβάλλον
εκτέλεσης της εφαρμογής σας, το αντίστοιχο αρχείο θα πρέπει να τοποθετείται σε σημείο του
οποίου η θέση είναι καθολική σταθερά (hardwired). Σε περίπτωση επιτυχίας, επιστρέφεται
κωδικός 200 διαφορετικά επιστρέφεται το αντίστοιχο αντικείμενο error log.
3. /admin/addpoints: Υποστηρίζει τη μέθοδο POST και προβαίνει σε προσθήκη νέων σημείων
φόρτισης που περιέχονται σε αρχείο csv που μεταφέρεται ως content type: multipart/formdata και mime type: 'text/csv'. Ο μορφότυπος του αρχείου πρέπει υποχρεωτικά να είναι ο
ίδιος με αυτόν που χρησιμοποιείται στην κλήση /admin/resetpoints.

ΣΕΛ 2

Λειτουργία
Ακολούθως δίνονται 7 endpoints για την επιτέλεση των βασικών λειτουργιών, τα οποία πρέπει όλα να
υλοποιηθούν ακριβώς όπως δίνονται, ώστε να μπορεί να γίνει αυτοματοποιημένος έλεγχος κατά την
παρουσίαση της εργασίας. Μπορείτε να προσθέσετε επιπλέον δικά σας endpoints, όπως απαιτείται από τις
περιπτώσεις χρήσης ή/και από τον τρόπο υλοποίησης. Για όλες τις κλήσεις ισχύουν τα ακόλουθα:
•

date/time format: οι ημερομηνίες που δίνονται ως path parameters (:from, :to) πρέπει να έχουν τη
μορφή "YYYYMMDD", ενώ οι χρονικές αναφορές (timestamps) που επιστρέφονται πρέπει να είναι της
μορφής "YYYY-MM-DD hh:mm". Βλ. πρότυπο IEEE 8601.

•

status enumeration: οι επιτρεπτές καταστάσεις των σημείων φόρτισης είναι οι ακόλουθες: available,
charging, reserved, malfunction και offline. Οι 4 πρώτες υποδηλώνουν ενεργή δικτυακή
επικοινωνία με το σημείο φόρτισης.

ΣΕΛ 7

Command Line Interface - CLI
Οι προδιαγραφές του CLI περιλαμβάνουν κλήσεις οι οποίες είναι αντίστοιχες με εκείνες του REST API και
μπορούν να εκτελεστούν μέσω shell, διαθέσιμου μόνο από την κονσόλα (command line, ssh) του
συστήματος που φιλοξενεί την εφαρμογή.

Λειτουργία
Το CLI θα υποστηρίζει τον μορφότυπο JSON και τον μορφότυπο CSV, όπως ακριβώς και στο REST API. Η
επιλογή του μορφότυπου θα καθορίζεται με παράμετρο όπως αναφέρεται παρακάτω. Τα αποτελέσματα
που επιστρέφει το CLI θα είναι ταξινομημένα ως προς το timestamp με φθίνουσα τάξη για τις περιπτώσεις
όπου επιστρέφεται πεδίο ημερομηνίας - from, ή το id σημείου για τις άλλες περιπτώσεις.
Το CLI θα καλείται από τη γραμμή εντολών με κλήσεις της μορφής:
$ se25XX scope --param1 value1 [--param2 value2 ...] --format fff

Όπου XX το αναγνωριστικό της ομάδας σας και scope όπως αναφέρεται στον πίνακα που ακολουθεί. Σε
περίπτωση που δεν δίνονται παράμετροι, θα πρέπει να εμφανίζονται οι υποστηριζόμενες από το CLI
παράμετροι για όλα τα scopes. Σε περίπτωση που δεν δίνεται η παράμετρος --format, η default τιμή για το
CLI είναι csv.
Για παράδειγμα, η κλήση για τη δέσμευση του σημείου φόρτισης 123 για 60 λεπτά είναι η ακόλουθη:
$ se24XX reserve --id 123 --minutes 60

Θα πρέπει να υποστηρίζονται οι ακόλουθες παράμετροι ανά scope.
